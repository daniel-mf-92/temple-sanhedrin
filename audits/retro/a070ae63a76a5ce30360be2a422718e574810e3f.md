## Findings

### WARNING-1: Disk reclaim uses broad recursive deletion without partition-boundary proof

Applicable laws:
- Law 9: Resource Supremacy / Crash on Log Failure
- Law 10: Immutable OS Image

Evidence:
- `Kernel/BookOfTruth.HC:2537-2567` adds `BotPurgeMask`, which recursively scans and deletes files from a caller-supplied mask.
- `Kernel/BookOfTruth.HC:2578-2582` builds `home_mask` from `blkdev.home_dir` and purges it as "user files first".
- `MODERNIZATION/MASTER_TASKS.md:213-219` requires user data, Book-of-Truth logs, and LLM models to live on a separate writable partition from the immutable OS image.

Assessment:
The commit does not prove that `blkdev.home_dir` is always constrained to the separate writable user partition. If `home_dir` can point at, alias, or traverse into the OS partition on a legacy layout, disk pressure could delete non-log files from the wrong trust boundary. This is a warning rather than a critical finding because the code tries to spare Book-of-Truth paths and this audit did not execute the guest.

Remediation:
- Add a reclaim root guard that verifies the target drive/partition is the writable user-data partition before any recursive purge.
- Record the drive/partition identifier in the Book-of-Truth reclaim event payload.

### WARNING-2: Keep-filter is substring-based and does not prove all log artifacts are protected

Applicable laws:
- Law 3: Book of Truth Immutability
- Law 9: Resource Supremacy / Crash on Log Failure

Evidence:
- `Kernel/BookOfTruth.HC:2524-2535` preserves paths only when the string contains `BookTruth`, `BookOfTruth`, or `Sefer`.
- `Kernel/BookOfTruth.HC:2557-2560` deletes every matched path that fails that substring check.
- `MODERNIZATION/MASTER_TASKS.md:193-197` says the log disk allocation is the last thing freed and the kernel must halt if the log cannot be written after reclamation.

Assessment:
The keep-filter protects current naming conventions but not a canonical ledger allocation, inode, directory root, or event type. A renamed log file, lowercase path, host-created serial capture mirror, or future log directory outside those substrings could be treated as expendable. The law is allocation/semantics based, not substring based.

Remediation:
- Replace substring keep logic with a canonical Book-of-Truth root/allocation table or a typed file attribute.
- Add a negative fixture proving names without `BookTruth` but marked as log storage are never deleted.
