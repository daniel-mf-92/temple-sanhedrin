### WARNING-002: Smoke script writes source-tree pycache unless the caller redirects it

Applicable laws:
- Law 5, No Busywork / validation evidence quality
- Law 10 by analogy for immutable-image discipline, limited to host tooling behavior

Evidence:
- `automation/bookoftruth-qemu-contract-area-summary-smoke.sh` runs `python3 -B -m py_compile "$REPO_DIR/automation/bookoftruth-qemu-contract-area-summary.py"`.
- In this environment, running the smoke directly failed with `Operation not permitted` while trying to write `automation/__pycache__/bookoftruth-qemu-contract-area-summary.cpython-314.pyc...`.
- Re-running with `PYTHONPYCACHEPREFIX=/tmp/templeos-gpt55-pycache` passed: `[bookoftruth-qemu-contract-area-summary-smoke] PASS`.
- The Makefile syntax-smoke path sets `PYTHONPYCACHEPREFIX`, but the smoke script itself does not, so direct invocation and harness contexts that do not set the variable can dirty or fail the source worktree.

Assessment:
The commit claims host-only smoke coverage and avoids QEMU, but its smoke is not cleanly non-mutating by default. This is a validation hygiene warning rather than a core Law 1/2 violation. It matters because the active TempleOS loop already carries generated `automation/__pycache__` worktree noise; repeating that pattern makes retroactive commit review and dirty-tree triage harder.

Recommended remediation:
- Set `PYTHONPYCACHEPREFIX="${PYTHONPYCACHEPREFIX:-$TMP_DIR/pycache}"` inside the smoke script before any `py_compile` call.
- Alternatively replace the direct `py_compile` call with an import-free syntax parse that writes only under the script's temp directory.
