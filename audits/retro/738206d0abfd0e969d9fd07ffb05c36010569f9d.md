## Findings

### WARNING: North Star preflight enforces air-gap evidence but not immutable-drive evidence

- Laws implicated: Law 10 Immutable OS Image, Law 2 Air-Gap Sanctity.
- Evidence: `automation/ns-preflight.sh` checks only that `automation/north-star-e2e.sh` contains `-nic none`. The QEMU command still conditionally adds a writable `-drive "file=$TMP_SHARED_IMG,format=raw,if=ide"` for the shared image, and there is no preflight assertion for `readonly=on` on any OS-image `-drive` path if the harness later moves from `-cdrom` to disk boot.
- Risk: the current ISO boot is read-only by virtue of `-cdrom`, and the shared image is not the OS image. The gap is still material because the preflight now owns North Star host assertions and can pass a future disk-backed QEMU launch that satisfies air-gap policy while missing Law 10 immutable-drive evidence.
- Suggested remediation: extend `automation/ns-preflight.sh` to distinguish shared-data drives from OS-image drives and reject any OS-image `-drive` stanza lacking `readonly=on`. Keep `-nic none` as a separate mandatory check.
