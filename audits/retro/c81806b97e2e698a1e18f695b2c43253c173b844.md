## Findings

### WARNING-001: North Star runner failures are recorded as green

The new `capture_snapshot` and `assert_snapshot` paths use:

```sh
if ! run_ns "$run_log"; then
  ns_rc=$?
fi
```

Inside a `! command` branch, `$?` is the status of the negation, not the failed command, so `ns_rc` becomes `0` after `run_ns` fails. A `/tmp` fixture runner that wrote a nonempty serial log and exited `7` produced `north_star_status=green`, `north_star_rc=0`, and script exit `0`.

Laws implicated: Law 5 / North Star Discipline.

Impact: a RED North Star run can be persisted as a green regression snapshot, masking the very signal WS1-04 is meant to preserve.

Suggested remediation: capture `run_ns` status without `!`, for example `run_ns "$run_log"; ns_rc=$?`, while guarding `set -e` explicitly.

### WARNING-002: Empty Book-of-Truth evidence compares as PASS

`extract_bot_lines` intentionally tolerates no matches with `|| true`, and `compare_snapshot` then only checks `cmp -s` on the extracted `.bot` files. Two serial logs with no `BoT:` lines compare as PASS because both extracted files are empty.

Laws implicated: Law 3 Book of Truth immutability, Law 8 Book of Truth immediacy/proximity evidence, and Law 5 North Star Discipline.

Impact: the harness can certify equality without proving that any Book-of-Truth event was captured, ordered, or preserved.

Suggested remediation: require a nonzero expected cardinality, ideally exactly the required North Star BoT sequence, before comparing equality.

### WARNING-003: `INPUT_LOG` bypass is not validated against North Star requirements

`run_ns` returns `0` immediately after copying `INPUT_LOG`, without requiring the input log to contain `BoT: boot ok`, `BoT: keypress=q`, and `BoT: halt clean` in order. Combined with the empty-compare behavior, fixture mode can create green manifests with `bot_lines=0`.

Laws implicated: Law 5 / North Star Discipline.

Impact: the validation command recorded in `MASTER_TASKS.md` uses `INPUT_LOG=/tmp/ns-fixture.log`, so the commit's own validation path depends on a bypass that can pass without the claimed North Star evidence.

Suggested remediation: route both live and `INPUT_LOG` modes through the same ordered-line validator before writing green status or accepting an assertion.
