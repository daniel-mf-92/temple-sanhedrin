## Findings

### Finding CRITICAL-001: Exported tamper policy API can disable sealed-page fail-stop

Laws implicated:
- Law 3: Book of Truth immutability
- Law 9: Resource supremacy / crash on log failure

Evidence:
- `Kernel/BookOfTruth.HC:625` introduces mutable `bot_tamper_halt_on_sealed=TRUE`.
- `Kernel/BookOfTruth.HC:673` declares `BookTruthTamperPolicySet(Bool halt_on_sealed=TRUE)`.
- `Kernel/BookOfTruth.HC:2252-2258` implements the setter and assigns `bot_tamper_halt_on_sealed=halt_on_sealed`.
- `Kernel/KExts.HC:1523` exports the setter to callers.

Assessment:
The default value is safe, but the exported setter accepts `FALSE`. Law 9 forbids any config flag, boot parameter, or API that disables halt-on-failure behavior. A sealed Book-of-Truth page fault is directly in the log integrity path, so allowing callers to turn the panic behavior off is a critical violation.

Required closure:
- Remove the external ability to set `halt_on_sealed` false.
- If an API remains, make it clamp to `TRUE` and record attempted disable requests as blocked tamper-policy events.

### Finding CRITICAL-002: Sealed-page panic is conditional after the commit

Laws implicated:
- Law 3: Book of Truth immutability
- Law 9: Resource supremacy / crash on log failure

Evidence:
- `Kernel/BookOfTruth.HC:2232-2234` copies `bot_tamper_halt_on_sealed` and increments `bot_tamper_fault_continue` when a sealed fault occurs while halt is disabled.
- `Kernel/BookOfTruth.HC:2248-2249` panics only when `event_type==BOT_EVENT_SEAL_FAULT && halt_on_sealed`.
- Before this commit, the panic condition was `event_type==BOT_EVENT_SEAL_FAULT` with no policy gate.

Assessment:
The commit changes sealed-page tamper response from unconditional fail-stop to conditional fail-stop. Law 9 requires the halt to be in the log write/failure path itself and not be removable by a separate policy handler. The new condition creates an execution path where a sealed log-page tamper fault is recorded, printed, and then execution continues.

Required closure:
- Restore unconditional panic on `BOT_EVENT_SEAL_FAULT`.
- Keep counters/status as observational telemetry only, not as a control that can change fail-stop behavior.

### Finding WARNING-003: The smoke test enshrines the unsafe continue path

Laws implicated:
- Law 3: Book of Truth immutability
- Law 9: Resource supremacy / crash on log failure

Evidence:
- `automation/bookoftruth-tamper-policy-smoke.sh:32-33` requires the policy setter to exist.
- `automation/bookoftruth-tamper-policy-smoke.sh:36-39` requires both the conditional halt gate and `bot_tamper_fault_continue++`.

Assessment:
The smoke test does not merely check a safe default. It makes the conditional panic gate and continue counter part of the accepted contract. That can cause future audits and CI checks to preserve a Law 9 violation instead of removing it.

Required closure:
- Change the smoke to require blocked-disable semantics, for example a strict setter that always leaves halt enabled and increments a blocked-disable counter.
- Add a negative assertion that no `BOT_EVENT_SEAL_FAULT` path can bypass `Panic`.
