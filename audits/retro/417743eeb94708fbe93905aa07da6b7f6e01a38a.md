## Findings

### CRITICAL: Build attestation can mutate trust-gate evidence without ledger emission

Laws implicated: Law 3 (Book of Truth immutability), Law 8 (Book of Truth immediacy and hardware proximity), WS14 secure-local gate semantics.

Evidence:

- `Kernel/BookOfTruth.HC:13184` adds `public Bool BookTruthModelBuildSet(I64 model_id,U64 build_hash,U64 bless_hash, U64 kernel_hash,Bool emit_event=TRUE)`.
- `Kernel/BookOfTruth.HC:13210-13215` writes `build_ok`, `build_mask`, `build_hash`, `build_bless_hash`, `build_kernel_hash`, and `build_tsc`.
- `Kernel/BookOfTruth.HC:13222-13227` wraps the only Book-of-Truth append in `if (emit_event)`.
- `Kernel/BookOfTruth.HC:13267-13275` allows secure-local promotion when `build_tsc!=0` and `build_ok` is true.

Impact:

Any caller that passes `emit_event=FALSE` can set the build attestation fields that satisfy `BOT_MODEL_GATE_BUILD` without producing a corresponding Book-of-Truth event. That creates an unledgered path from quarantine-gate evidence mutation to trusted promotion eligibility. For a secure-local attestation gate, the ledger emission should be mandatory, synchronous, and inseparable from the state transition.

Recommended remediation:

Remove the public bypass for this gate. `BookTruthModelBuildSet` should always append either `BOT_EVENT_NOTE` or `BOT_EVENT_VERIFY_FAIL` before returning after state mutation, and the smoke check should assert there is no `emit_event` parameter on the build-attestation setter.
