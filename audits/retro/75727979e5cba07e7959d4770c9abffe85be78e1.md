- **Evidence:** `BookTruthModelImport`, `BookTruthModelVerify`, and `BookTruthModelPromote` expose `Bool emit_event=TRUE` in public signatures and only call `BookTruthAppend(...)` when that caller-controlled flag is true.
- **Locations:** `Kernel/BookOfTruth.HC:12389`, `Kernel/BookOfTruth.HC:12429`, `Kernel/BookOfTruth.HC:12439`, `Kernel/BookOfTruth.HC:12464`, `Kernel/BookOfTruth.HC:12478`, `Kernel/BookOfTruth.HC:12500`, `Kernel/BookOfTruth.HC:12516`, `Kernel/KExts.HC:103`, `Kernel/KExts.HC:106`, `Kernel/KExts.HC:108`
- **Why it violates:** LAWS.md forbids any flag, config, or API that disables logging. These are public kernel APIs for security-relevant model import, verification, and promotion state changes, so `emit_event=FALSE` is a direct bypass path for the Book of Truth record.
- **Required remediation:** Remove the public suppression parameter from these APIs. If internal tests need fixture-only suppression, keep it in a private helper that cannot mutate trusted model state without logging.

- **Evidence:** Import stores `sha_hi`, `sha_lo`, and `tok_hash` in mutable `bot_models[idx]`, but the immutable payload only encodes marker/model/quant/provenance/slot. Verify and promote payloads similarly encode marker/model/slot/ok without the expected or observed hash material.
- **Locations:** `Kernel/BookOfTruth.HC:12415`, `Kernel/BookOfTruth.HC:12424`, `Kernel/BookOfTruth.HC:12455`, `Kernel/BookOfTruth.HC:12460`, `Kernel/BookOfTruth.HC:12512`
- **Why it matters:** The Book of Truth record is the canonical historical witness. A later auditor cannot reconstruct which model bytes or tokenizer hash were imported or promoted from the immutable log alone, and the status command reads mutable RAM state instead of sealed ledger evidence.
- **Required remediation:** Define a multi-entry ledger encoding or dedicated event schema that synchronously records full model hash, tokenizer hash, quant type, provenance, verification result, and promotion result in the append-only chain.
