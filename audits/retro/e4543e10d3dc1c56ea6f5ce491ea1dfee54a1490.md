## Findings

### CRITICAL: Deterministic gate can mutate trusted-promotion state without emitting a Book-of-Truth event

Evidence:
- `BookTruthModelDetRun(...)` is a public HolyC API with caller-controlled `Bool emit_event=TRUE`.
- The function updates `det_ok`, `det_seed`, `det_prompt_hash`, `det_window_hash`, `det_baseline_hash`, and `det_tsc` before the `if (emit_event)` block.
- With `emit_event=FALSE`, the deterministic gate can satisfy the later `secure-local` promotion condition without any `BOT_MODEL_MARK_DET` ledger event.

Law impact:
- Law 3 forbids any API that can disable logging.
- Law 8 requires Book-of-Truth records synchronously and close to the operation being recorded.
- This is not merely a status-print omission; it lets a gate-relevant model state transition become invisible to the ledger.

Recommended remediation:
- Remove the public `emit_event` parameter from `BookTruthModelDetRun`.
- Always emit either `BOT_EVENT_NOTE` or `BOT_EVENT_VERIFY_FAIL` for every deterministic gate attempt.
- If silent fixture setup is needed for host tests, keep it outside core HolyC state mutation paths.

### WARNING: Deterministic gate event payload records only 7 bits of `seed`

Evidence:
- `BookTruthModelDetRun` stores full `det_seed=seed`, but the Book-of-Truth payload encodes only `((seed&0x7F)<<8)`.
- WS14-05 is marked complete as "fixed prompt/seed/logit-window parity vs blessed baseline".
- The committed smoke harness checks for the function and status strings, but not seed round-trip or payload decode fidelity.

Law impact:
- Law 5 North Star Discipline: the task claims deterministic parity evidence, but the canonical ledger event cannot reconstruct or distinguish seeds outside the low 7 bits.
- This weakens historical auditability of deterministic inference evidence.

Recommended remediation:
- Encode the full seed in either the event payload, an adjacent sealed ledger entry, or a deterministic side record that remains hash-chain-bound.
- Add a decode/status check that proves two seeds differing above bit 7 remain distinguishable in Book-of-Truth evidence.

### WARNING: Secure-local promotion still allows success without a successful parser gate run

Evidence:
- `BookTruthModelPromote` gates parse failures only when `parse_fmt!=0 && !parse_ok`.
- If no parser gate was run, `parse_fmt==0` and promotion can still pass when schema, verify, and deterministic parity are satisfied.
- `MODERNIZATION/MASTER_TASKS.md` marks WS14-04 and WS14-05 complete, so secure-local promotion now implies both parser hardening and deterministic inference gates are expected policy components.

Law impact:
- Law 5 North Star Discipline: the secure-local model-promotion story is incomplete if a trusted promotion can skip one of the completed hardening gates.
- No direct air-gap or HolyC-purity breach was found.

Recommended remediation:
- Require `parse_fmt!=0 && parse_ok` for `secure-local` promotion, or explicitly document parser-gate optionality in WS14 and gate status output.
- Extend `automation/bookoftruth-model-det-smoke.sh` or the model-gate smoke to assert the no-parse promotion failure path.
