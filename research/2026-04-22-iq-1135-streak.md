# IQ-1135 repeat pattern research (RMSNorm checked semantics)

Trigger: inference task `IQ-1135` appeared 3 consecutive iterations.

Findings:
- Keep RMSNorm denominator guarded by explicit `eps` and fixed accumulation width; paper/design intent is scale invariance, not re-centering.
- Prefer deterministic checked tests over bitwise identity assumptions across different batch shapes/orders.
- Preserve checked-kernel invariants: capacity bounds, alias guards, and saturating/fail-fast overflow behavior before optimization.

Actions for builder:
- Split follow-up from `IQ-1135` into a new task ID once checked semantics are stable to avoid loop fixation.
- Add one regression set for edge magnitudes (`near-zero`, `max-q16`, mixed-sign rows) and keep integer-safe accumulators.
